Construcción de Software

La creación detallada y funcional de un producto de software mediante un conjunto específico de actividades

Creación Detallada y Funcional

  • Énfasis en la creación detallada de software significativo y funcional
  • Combinación de actividades: análisis, diseño, codificación, verificación y pruebas
  • Requiere conocimientos de algoritmos y prácticas de codificación
  • Fundamental para el éxito del producto final

Área de Conocimiento

  • Parte del Cuerpo de Conocimiento Central (CBOK) del GSwE2009
  • Una de las 11 Áreas de Conocimiento en Ingeniería de Software
  • Definida como la creación de software significativo y funcional
  • Incluye: codificación, verificación, pruebas unitarias, pruebas de integración y depuración

Fase de Programación

  • Convierte el diseño en código funcional
  • Desarrolladores escriben, prueban y depuran el software
  • Utilización de lenguajes de programación y herramientas seleccionadas
  • Construcción de componentes según especificaciones de diseño
  • Requiere atención al detalle y colaboración constante
  • Culmina con software ejecutable listo para pruebas

Proceso de Construcción de Software

Análisis y Diseño
Definición de requisitos y diseño de la arquitectura del software
Codificación
Implementación del diseño en código utilizando lenguajes de programación
Verificación
Revisión del código y verificación de su correcto funcionamiento
Pruebas
Pruebas unitarias y de integración para asegurar la calidad
Depuración
Corrección de errores y problemas identificados
